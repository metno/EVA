eva.config¶

Configuration module, containing classes necessary for automatic configuration of objects from a configuration file.

class eva.config.ConfigurableObject[source]¶

Base class that allows the subclass to define configuration options that can be populated from an INI file, along with a list of required and optional configuration variables.

In order to use configuration classes in your EVA configuration, they MUST be derived from this class. Optionally, they may reimplement the _factory() method, which must return a configured class instance of any type.

Usage:

incubator, object_ = eva.config.ConfigurableObject().factory(
    {
        'foo': '1, 2, 3, 5, 7, 11',
        'bar': 'baz',
    }
)

# `incubator` is a reference to the ConfigurableObject instance, while
# `object_` is a reference to the object returned by _factory().
# Normally, you proceed with calling init() if your instantiated class
# is a ConfigurableObject instance:

object_.init()

Configurable variables are defined with the CONFIG dictionary.

class MyClass(ConfigurableObject):
    CONFIG = {
        'foo': {
            'type': 'list_int',
            'default': '1,2,3',
        },
        'bar': {
            ...
        },
    },

The types are defined as normalize_config_<type> functions in this class. Subclasses might implement their own normalization functions as needed. A list of provided configuration normalizers are provided below:

  • normalize_config_string()
  • normalize_config_int()
  • normalize_config_positive_int()
  • normalize_config_float()
  • normalize_config_null_bool()
  • normalize_config_bool()
  • normalize_config_list()
  • normalize_config_list_string()
  • normalize_config_list_int()
  • normalize_config_config_class()

To define configuration options as required or optional, you may do:

class MyClass(ConfigurableObject):
    REQUIRED_CONFIG = ['foo']
    OPTIONAL_CONFIG = ['bar']

Note that variables that are not in any of these two lists will be regarded as invalid options, and trying to use them will throw an exception.

The normalized variables are made available in the env class member. For instance:

assert object_.env['foo'] == [1, 2, 3]  # True

_factory()[source]¶

This method is called after load_configuration(), so you can use the env dictionary for your object configuration.

load_configuration(config)[source]¶

Normalize input configuration based on the configuration definition: split strings into lists, convert to types.

Parameters:

config (dict) – dictionary with configuration variables.
Raises:
  • eva.exceptions.InvalidConfigurationException – when a configuration value is invalid (normalizer function raised an exception).
  • eva.exceptions.MissingConfigurationException – when a configuration value defined in REQUIRED_CONFIG is missing from the configuration dictionary.
  • RuntimeError – when a configuration variable is missing from the CONFIG hash, or has defined a normalization function that is not defined as a class member. This error indicates a bug in a subclass.

eva.config.resolved_config_section(config, section, section_keys=None, ignore_defaults=False)[source]¶

Recursively pull in includes and defaults for a configuration section, and combine them into a single dictionary.

This function provides infinite recursion protection, guaranteeing that a single configuration section will be read only once.

Parameters:
  • section (configparser.ConfigParser) – reference to a configuration parser.
  • section_keys (list) – list of section keys already parsed, used for infinite recursion protection.
  • ignore_defaults (bool) – whether or not to include the ‘defaults.<SECTION-BASENAME>’ configuration section.
Raises:

eva.exceptions.MissingConfigurationSectionException – when the configuration section does not exist.
Return type:

dict
Returns:

dictionary of configuration values.
